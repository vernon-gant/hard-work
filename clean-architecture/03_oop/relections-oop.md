In my OOP solution I used more ADTs and split the command execution from its parameter creation and validation. I just wanted to add validation for incoming parameters, that's why I added the `ParameterProvider`. There were only 2 CORE objects - Robot and Cleaning Device which could move and stat/stop cleaning. If we would have more rules, I would encapsulate the state of start/stop into separate classes and build some sort of a state machine. The disadvantage I see is that printing to the console was done from the commands directly and not using the `transfer_to_cleaner`, this is actually bad. But I could reduce the amount of `if/else` chains by encapsulating commands into separate objects!
